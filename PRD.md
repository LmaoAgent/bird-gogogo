## 1. 产品概述

### 1.1 一句话定位
控制一支「蒜鸟大军」在桥上狂奔，沿途吃加成门滚雪球式扩军，撞碎发霉腐烂的蒜系军团与 BOSS——魔性、解压、易上手、强传播的微信休闲小游戏。

### 1.2 立项理由
- **品类被验证**：Count Masters / Army Runner 类玩法在超休闲市场长期跑量稳定，规则一眼看懂、单局 30–90 秒、天然适合竖屏短视频素材。
- **差异化靠 IP**：以「蒜鸟」这一自带传播属性的魔性形象替换通用小人，满屏蒜鸟乌泱泱冲锋的画面本身就是买量素材和社交话题；「新鲜蒜鸟 vs 发霉军团」的主题自洽又有梗。
- **变现清晰**：休闲品类以激励视频为现金牛，叠加去广告礼包 + 皮肤内购拔高 ARPU。

### 1.3 目标用户
- 泛休闲玩家，微信生态下沉与广谱人群，年龄跨度大（12–45 为主）。
- 碎片时间解压、爱晒战绩、易被短视频/朋友圈素材吸引的用户。
- 核心画像：不追求硬核成长，追求「爽快滚雪球 + 通关正反馈 + 和好友比一比」。

### 1.4 产品目标与北极星指标
- 北极星：**广告观看人均次数（IPU）× DAU** 驱动的 IAA 收入，叠加 IAP 付费率。
- 首发验证目标【建议】：次留 ≥ 40%、7 留 ≥ 15%、人均单日局数 ≥ 8、激励视频渗透率 ≥ 50%、IPU ≥ 4。
- 商业化目标【建议】：LTV / CPI ≥ 1.2（买量可放量的最低盘），IAP 付费率 2%–4%。

## 2. 竞品与差异

| 维度 | 通用竞品（Count Masters / 人海冲冲冲类） | 本作差异 |
|---|---|---|
| 主角 | 通用火柴人/士兵 | 蒜鸟大军，形象魔性、素材辨识度高 |
| 敌人 | 红巨人 / 敌军 | 蒜系反派（霉菌怪、烂蒜王），主题自洽 |
| 玩法 | 加成门累加 + 撞击 | 沿用被验证核心，微创新在「门」的花样与主题包装 |
| 社交 | 多为纯榜单 | 微信好友异步榜 + 分享裂变（晒战绩/求复活） |

**结论**：核心玩法不重复造轮子（照搬被验证的手感），资源集中在 IP 包装、素材传播、门的花样设计与数值手感调优上。

## 3. 核心玩法设计

### 3.1 核心循环
```
开局出兵（N0 只蒜鸟）
   → 左右滑动走位，吃加成门 / 躲陷阱门（滚雪球扩军）
   → 撞击敌方军团洪流（兵力对撞消耗其血量）
   → 击穿 BOSS → 通关结算（金币 + 星级 + 可看广告翻倍）
   → 失败可看激励视频原地复活续冲
   → 升级/换皮肤 → 下一关
```

### 3.2 操作方式
- **单指左右拖动**控制大军整体横向位置（前进自动）。手感是全部体验的地基，P0 必须先打磨到位。
- 无跳跃、无攻击键——零学习成本。

### 3.3 加成门系统（玩法花样的核心载体）
沿赛道两侧/中间布置各类「门」，玩家用走位选择吃哪一侧：

| 门类型 | 效果 | 作用 |
|---|---|---|
| 蓝色 `+1` 连排门 | 逐格 +1，一长串累加 | 基础扩军，引导走位贴边 |
| 黄色 `+99` 大门 | 一次性大额加兵 | 稀有/需冒险绕路的高回报 |
| 乘法门 `×2 / ×3` | 当前兵力翻倍 | 滚雪球关键点，越晚吃越赚 |
| 减法/除法陷阱门 `-50 / ÷2` | 扣兵 | 惩罚走错侧，制造抉择 |
| 双选门 | 左右各一门二选一（如 左`×2` vs 右`+80`） | 核心策略抉择点 |
| 动态门 | 数值滚动跳变 / 门左右移动 | 增加操作与判断难度 |

【建议】以「双选门」制造每关 3–5 个关键决策，是留存与「差一点就过」挫败感/再来一局动机的来源。

### 3.4 战斗（对撞）机制
截图中敌方带血条（如 700→181），确认为**持续对撞消耗模型**，而非一次性碰撞：

- 我方队伍规模 `N`，敌方军团/BOSS 有血量 `H`。
- 对撞时双方按速率互相消耗：敌方血量下降速率 ∝ 我方存活数 × 单兵 DPS `d`；我方每秒损失 ∝ 敌方反伤系数 `r`。
- **胜负判定**：撑到 `H = 0` 且 `N > 0` → 突破；`N` 先归零 → FAIL。

**可落地的关卡数值方法【建议】**（让每关设计可验证）：
1. 定义「兑换比」`k` = 1 只蒜鸟能抵消的敌方血量（初期 `k` 较高、随关卡下降以提难度）。
2. 过关最小兵力 `N_min ≈ H / k`。
3. 关卡设计流程：先定敌方血量 `H` 与目标难度 → 反推 `N_min` → 布置门的总加成，使**最优路线兵力 ≥ N_min、次优路线 < N_min**（制造「走对才过」的抉择）。
4. 验收标准：每关存在一条「贪心吃门」通不过、需要正确取舍才过的路径。

### 3.5 关卡结构与难度曲线
- **第 1–3 关（新手）**：无陷阱门、`H` 远低于最优兵力，保证必过，教会「贴门走位」与「滚雪球」。
- **第 4–10 关**：引入乘法门、单个陷阱门。
- **第 11 关起**：双选门、分叉路、动态门、移动障碍逐步叠加。
- **BOSS 节奏**：每 5 关一个小 BOSS（烂蒜精英），每 20 关一个大 BOSS（烂蒜魔王，多阶段/高血量）。
- **首发 40 关**（已定），后续版本持续追加；关卡耗尽后进入「无尽/循环+难度递增」模式保证长线可玩。

### 3.6 失败与复活
- FAIL 后弹出**激励视频复活**：看广告原地保留当前兵力/半血续冲（每局限 1–2 次，之后只能重开）。这是核心广告位与「差一点」情绪的变现出口。

## 4. 角色与美术

### 4.1 蒜鸟大军（主角）
- 形象特征：白胖蒜瓣身体 + 鸟嘴 + 小翅膀 + 魔性/呆萌表情；集群冲锋时乌泱泱一片，喜感与压迫感并存。
- 皮肤/变体【建议】（外观为主，可挂极小数值或纯外观，避免 P2W 破坏公平）：
  - 独头蒜（金色·稀有）、黑蒜（暗黑）、糖蒜（Q 萌）、发芽蒜（搞笑）、腊八蒜（节日）等。
- 皮肤是内购与激励视频（碎片）的核心供给。

### 4.2 蒜系反派（敌人 + BOSS）
- 世界观：**新鲜 vs 腐烂**——蒜鸟大军讨伐发霉变质的「霉烂军团」。
- 杂兵：霉菌怪（绿毛）、蒜蛆虫、洋葱刺客（催泪 debuff 可选）。
- BOSS：烂蒜魔王（发黑流脓的巨型蒜，对应截图红巨人位），多阶段血条。
- 主题好处：叙事自洽、契合「蒜」梗、便于做「保鲜/防霉」类关卡机制与活动。

### 4.3 场景与 UI
- 场景（已定）：延续「桥/赛道」结构，**首发主题＝菜市场大桥**；腌菜坛通道、冷链冰库作为后续换肤主题。
- 美术风格【建议】：卡通魔性、高饱和、大对比，适配短视频素材与小屏识别。
- 渲染【建议】：2.5D（伪 3D）或轻量 3D，优先控制包体与低端机性能。

> **IP 版权（已定）**：形象来源＝**AI 生成 + 原创化**。蒜鸟是网络泛化 meme，但部分具体画风/表情包有原作者，商用必须**原创化设计**自有蒜鸟形象——自定特征比例、配色、表情规范，**避开与特定作者作品高度相似**，并留存设计过程记录以备权属证明。

## 5. 养成与数值系统

保持轻养成（休闲品类不宜重成长）：

| 系统 | 内容 | 货币 |
|---|---|---|
| 起始兵力升级 | 提升开局 `N0` | 金币 |
| 单兵强度升级 | 提升单兵 DPS `d` / 存活 | 金币 |

- **货币体系（已定）：首发单货币＝金币**。Plan B 首发无内购，钻石既没有获取来源、又与金币功能重复，徒增一个系统；待版号到位开内购时再引入钻石作为硬通货。

## 6. 商业化设计（IAAP 混合）

### 6.1 广告位设计（现金牛，激励视频为主）

| 广告位 | 类型 | 触发 | 频控【建议】 |
|---|---|---|---|
| 复活续冲 | 激励视频 | FAIL 后 | 每局 1–2 次 |
| 结算翻倍 | 激励视频 | 通关结算「金币 ×2/×3」 | 每次通关可选 |
| 开局增益 | 激励视频 | 开局「看广告 +50 起始兵力/带一次乘法 buff」 | 每局 1 次 |
| 免费宝箱/皮肤碎片 | 激励视频 | 主界面每日限次 | 每日 3–5 次 |
| 关卡插屏 | 插屏 | 通关/失败后（正反馈时插，避开挫败） | N 关一次，节流 |
| （可选）底部 Banner | 原生 banner | 主界面 | 谨慎使用，体验优先可不上 |

原则：**激励视频是主力**（玩家主动、体验正向、eCPM 高）；插屏节流；banner 慎用。

### 6.2 内购商品（拔高 ARPU）

| 商品 | 说明 | 价位【建议】 |
|---|---|---|
| 去广告礼包（核心） | 移除强制插屏/banner；**保留激励视频**（激励是玩家主动领奖，通常不移除） | ￥6–30 |
| 皮肤 / 蒜鸟形象 | 外观收集 | ￥6–68 |
| 永久起始兵力礼包 | 提升开局战力（注意别破坏关卡挫败感的核心循环） | ￥12–30 |
| 新手/首充礼包 | 首充双倍、超值引导包 | ￥6 |
| 月卡 | 每日领钻石 + 去广告体验 | ￥30/月 |

### 6.3 付费与广告的节奏
- 前期以激励视频建立「看广告=爽/得利」的正循环；
- 首充与去广告礼包作为最主要 IAP 转化点，在玩家第 1 次 FAIL 复活、第 1 次被插屏打断后适时弹出；
- 关键：**不把内购做成越过核心循环的付费墙**，否则休闲用户流失。

### 6.4 商业化指标
- IAA：IPU（人均广告次数）、eCPM、激励渗透率、广告 ARPU。
- IAP：付费率、ARPPU、去广告礼包转化率。
- 综合：LTV、回本周期、ROI（对买量至关重要）。

## 7. 社交与排行榜（微信好友异步榜）

- **实现方式**：微信开放数据域（`wx.getFriendCloudStorage` / `setUserCloudStorage`），无需自建服务器。
- **榜单维度【建议】**：历史最高关卡、历史最大兵力（单局峰值）。
- **展示位**：主界面入口 + 通关结算页「你超过了 X 位好友」。
- **分享裂变**：
  - 晒战绩卡片（带兵力峰值/关卡，诱导点击）；
  - 「挑战好友这一关」分享；
  - 每日首次分享得奖励（金币/免费复活）；
  - 【待确认，可选】求复活裂变：好友点你的分享帮你复活（拉新钩子，但要防刷）。
- **防作弊**：开放数据域数据前端可篡改，纯 IAA 阶段风险可接受；若上全服榜或与内购/奖励强挂钩，需服务端校验。

## 9. 技术方案

- **引擎【建议】**：Cocos Creator（微信小游戏生态最成熟、2.5D/3D 跑酷适配好、社区资源多）。备选 LayaAir。
- **包体控制**：微信小游戏首包 ≤ 4MB，采用分包加载 + 远程资源（CDN/微信云）加载皮肤与后续关卡。
- **微信能力**：开放数据域（排行榜）、激励视频/插屏广告组件、录屏分享、订阅消息（回流提醒）、虚拟支付（**需企业主体 + 版号**，见第 10 节）。
- **后端**：
  - **纯 IAA / MVP 阶段**：几乎无需自建后端。存档走微信云存储/本地 + 云备份，排行榜走开放数据域。
  - **开内购后**：需服务端校验微信支付回调、发货与订单幂等。**【建议】** 用微信云开发 CloudBase（免运维），避免自建服务器运维成本。
- **数据埋点**：接入统计（微信官方数据助手 + 第三方/自建），埋点覆盖：开局、每关进入/通关/失败、每类门吃取、复活点击/完成、广告请求/展示/完成、内购点击/成功、分享、留存漏斗。埋点是买量优化的命脉，P0 起同步建设。

## 10. 合规与上线（关键约束，务必先读）

> **这是本项目最大的现实约束与风险点，直接决定上线路径。**

### 10.1 主体与版号硬约束
- **微信小游戏虚拟支付（内购）要求企业主体，并在类目审核时提供游戏版号（ISBN）。个人主体小游戏不能开通虚拟支付（即不能做 IAP）。**
- 含内购的游戏通常需要：**企业营业执照 + 软件著作权 + 游戏版号 + 文网文（网络文化经营许可证）**。版号申办周期长、且需公司主体，是排期与成本的最大变量。
- 广告变现（流量主）：企业主体可较快开通；个人主体有流量/门槛限制。

### 10.2 上线路径建议（Plan A / B）
- **Plan A（有版号）**：企业主体 + 版号到位 → 首发即完整 IAAP（广告 + 内购）。
- **Plan B（无版号，先跑量）✅ 本项目采用**：先以**纯 IAA 版本**（仅广告、无内购）上线，快速验证留存与买量 ROI；待版号下来后，通过版本更新开启内购，升级为 IAAP。**节奏（用户决策）**：先上纯广告版验证可行性（留存 / ROI），**版号暂缓**；验证通过再启动版号 / 软著 / 文网文办理，避免版号沉没成本。
- 本 PRD 的功能已按「先 IAA 可独立成立、内购为可插拔增量」设计，兼容两条路径。

### 10.3 其它合规
- **防沉迷**：接入国家新闻出版署防沉迷实名认证（未成年时段/时长/充值限制）。
- **备案**：小程序/小游戏备案（工信部）+ 微信侧类目审核。
- **隐私**：用户隐私保护指引、隐私政策、最小化信息收集。
- **广告合规**：内容合规、频次不误导、未成年人广告限制。

## 11. 里程碑与 MVP 范围

| 阶段 | 目标 | 交付物 | 周期【建议】 |
|---|---|---|---|
| **P0 原型** | 验证手感 | 1 条赛道、蓝/黄/乘法门、对撞、胜负、10 关、可玩 demo | 3–4 周 |
| **P1 MVP** | 完整核心可上线（IAA） | **40 关** + 难度曲线、蒜鸟大军 + 基础皮肤、蒜系敌人 + 2 BOSS、激励视频复活/翻倍、结算、本地+云存档、新手引导、核心埋点 | 6–8 周 |
| **P2 广告+社交** | 广告变现与传播（纯 IAA） | 插屏广告、皮肤系统（金币/广告碎片解锁）、微信好友榜 + 定向挑战 PK、分享裂变、签到/每日任务 | 4–5 周 |
| **P3 上线买量 →（版号到位）升级 IAAP** | 放量 + 开内购 | 上架、AB 测试、买量素材、投放与数据迭代；**版号到位后**版本更新开启：去广告礼包、皮肤商城、月卡等内购 | 持续 |
| **P4 长线运营** | 留存与拓展 | 新皮肤、节日活动、赛季周榜、无尽模式、新玩法门；评估抖音小游戏多端 | 持续 |

> 周期按**单人 + AI 辅助**估算（已定团队配置）；`P0工程骨架.md` 里的 15 人天为纯开发粗估，不含美术生成与数值调参。

**MVP 判定标准**：核心手感达标 + 次留 ≥ 目标 + 激励视频渗透达标，再进入 P2/买量放量。

## 12. 关键指标与验证（Definition of Done）

| 目标 | 验证方式 |
|---|---|
| 手感达标 | P0 内部/小范围试玩，「再来一局」意愿强 |
| 留存达标 | 次留 ≥ 40%、7 留 ≥ 15% |
| 玩法有抉择 | 每关存在「贪心吃门通不过、需正确取舍才过」的路径 |
| 广告健康 | IPU ≥ 4、激励渗透 ≥ 50% |
| 商业可放量 | LTV/CPI ≥ 1.2 |
