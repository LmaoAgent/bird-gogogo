## 基于真实玩法修正的视觉基准

- 玩家蒜鸟位于屏幕底部，使用 P0 背面形象朝屏幕上方射击；验收高度为 52px。
- 怪群从赛道上方向屏幕下方冲锋，使用统一 35° 左右俯视正面视角；近景小怪高度 40px、远景 20px。
- 小怪单体只保留“深色圆团 + 亮色毛冠 + 两只脚”，集群排成连续怪潮，不按单体立绘密度使用。
- 维度门是低矮实心数值板，不是拱门或可穿过的门框；每侧可连续排 16 块。
- 四个正向维度门同形，仅以蓝 / 紫 / 橙 / 红区分；陷阱门使用黑红斜纹。
- 障碍使用低矮尖锐的黑 / 红 / 黄视觉；奖励油桶使用圆润直立的青蓝 / 白 / 金视觉。

真实玩法比例预览：`preview/p1_shooter_gameplay_density_preview.png`，其中包含 128 只小怪、80 颗子弹、24 只蒜鸟和每侧 16 块门。

## Cocos Creator 接入

1. 优先导入 `atlases/` 中对应的 PNG 与 `.plist`；若工程管线使用 JSON，则绑定同名 `.json`，不要重复绑定两份元数据。
2. 动画帧率、循环、additive 混合、pivot、@2x 九宫格 inset 和运行时参考尺寸见 `integration_manifest.json`。
3. `gate_barrier_body_base_01.png` 使用九宫格横向拉伸；`gate_barrier_stripe_tile_01.png` 的 X 轴 wrap mode 设为 Repeat。
4. 维度门正面数字由程序使用 P0 位图字体渲染；素材本身故意保持空白大色块。
5. 小怪群渲染按单个 base sprite 实例化，通过帧相位、少量缩放和位置扰动制造流动感；不要为 200 只怪准备 200 套单体细节。
6. 实机建议：近景蒜鸟 52px、小怪 40px、厚皮 72px；纵深方向插值缩小到小怪 20px、维度门 24px。
