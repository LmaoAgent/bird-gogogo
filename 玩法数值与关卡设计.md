# 《蒜鸟冲冲冲》玩法数值与关卡设计（细化 PRD 第 3 章）

> 面向程序 / 策划，可直接开工。核心原则：**一切数值走配置表、数据驱动、不硬编码**，便于策划调参与热更。

## 1. 坐标系与操作
- 竖屏，赛道沿 **Z 轴纵深**延伸；表现上世界向玩家滚动（或大军前进、镜头跟随）。
- **输入**：单指水平拖动 → 映射大军**队形中心 X**。无跳跃、无攻击键。
- 大军跟随手指做平滑：`centerX = lerp(centerX, targetX, followSmooth)`。
- 横向边界：赛道左右墙，队形中心 X 夹取（clamp）在赛道内。

## 2. 兵力与队形
- `N0` 起始兵力（可升级），`N` 实时兵力。
- **队形**：以中心点向外排布（圆盘 / 网格），阵型半径 `R ∝ √N`（数量越多摊越大）。
- **同屏渲染上限** `N_render`（建议 120–150）：超过部分不实例化，用**缩放 + 兵力数字**表现"更多"，保证低端机性能。
- 逻辑兵力 `N` 与渲染数解耦：`N` 可远大于 `N_render`。

## 3. 加成门 / 陷阱门数值

### 3.1 门配置结构（JSON schema 示意）
```json
{
  "id": "gate_0007",
  "type": "add | mul | sub | div | pick",   // 加/乘/减/除/双选
  "value": 99,                                // pick 时见 options
  "side": "left | center | right",
  "posZ": 120,                                // 赛道纵深位置
  "options": [                                // 仅 type=pick
    { "type": "mul", "value": 2, "side": "left" },
    { "type": "add", "value": 80, "side": "right" }
  ]
}
```

### 3.2 门效果
| type | 作用 | 说明 |
|---|---|---|
| add | `N += value` | 蓝 `+1` 连排 / 黄 `+99` |
| mul | `N *= value` | `×2/×3`，越晚吃越赚，滚雪球关键 |
| sub | `N -= value` | 陷阱，`N` 下限 0 |
| div | `N = floor(N/value)` | 陷阱 `÷2` |
| pick | 玩家走位选 options 之一 | 核心抉择门 |

- 蓝 `+1` 连排：以数组批量生成，引导玩家贴边一路吃。

## 4. 对撞战斗模型（核心）

### 4.1 变量
| 符号 | 含义 |
|---|---|
| `N` | 撞击瞬间玩家兵力 |
| `H` | 敌方军团 / BOSS 血量 |
| `k` | 兑换比：1 兵可抵消的敌方血量（关卡参数，随关卡下降以提难度） |
| `d` | 单兵 DPS（仅影响对撞表现时长） |
| `N_min` | 过关最小兵力 `= H / k` |

### 4.2 解耦设计（推荐）：结果由 k 决定，表现由 d 决定
- **结果判定**（可预测，策划靠 `H`/`k` 控难度）：
  - 撞击时 `N ≥ N_min` → **突破**，穿透后剩余兵力 `N' = N - H/k`（用于继续冲下一波 / 星级）。
  - `N < N_min` → 兵力被消耗至 0，**FAIL**。
- **表现演出**（有张力，不改结果）：对撞动画时长 `T = clamp(H / (N·d), T_min, T_max)`；期间双方数字滚动下降、蒜末/霉菌飞溅。`d` 只调观感，建议 `T_min=0.6s, T_max=2.5s`。

> 好处：难度可控（改 k/H 即可），表现可调（改 d），互不干扰。BOSS 多阶段 = 串联多个 H 段。

### 4.3 每帧结算（伪代码）
```
onContact(dt):
    if N >= H/k:                 // 本波可破
        playSmashAnim(T)         // 表现
        N = N - H/k              // 穿透损耗
        advanceToNextWaveOrWin()
    else:
        drainAnim()              // 兵力被吃光
        N = 0
        onFail()
```

## 5. 关卡配置与难度曲线

> **首发共 40 关**（已定）；下方给出前 10 关具体配置，11–40 关按同一难度曲线延展，打完进入无尽模式。

### 5.1 关卡配置结构（JSON schema 示意）
```json
{
  "level": 7,
  "trackLength": 900,
  "gates": [ /* 门数组，见 §3.1 */ ],
  "enemies": [ { "type": "mold", "H": 300, "posZ": 850 } ],
  "boss": null,
  "k": 4.0,
  "targetN": 90,          // 设计期望的最优路线兵力（用于校验 > N_min）
  "star": { "3": 0.6, "2": 0.3 }   // 剩余兵力比例阈值
}
```

### 5.2 前 10 关配置示例（N0 起始 = 10）
| 关卡 | 敌方 H | k | N_min=H/k | 最优路线兵力(目标) | 引入机制 | 设计意图 |
|---|---|---|---|---|---|---|
| 1 | 15 | 5.0 | 3 | ~30 | 蓝 `+1` | 教学，必过 |
| 2 | 30 | 5.0 | 6 | ~45 | 蓝 `+1` | 教学，必过 |
| 3 | 60 | 5.0 | 12 | ~60 | 黄 `+99` | 引导贴门 |
| 4 | 120 | 4.5 | 27 | ~80 | 乘法 `×2` | 学会"晚吃乘法" |
| 5 · 小BOSS | 250 | 4.0 | 63 | ~110 | 小 BOSS | 第一个坎 |
| 6 | 200 | 4.0 | 50 | ~120 | 分叉路 | 路线选择 |
| 7 | 300 | 4.0 | 75 | ~110 | 陷阱 `-50` | 学会避坑 |
| 8 | 400 | 3.5 | 115 | ~150 | 双选门 | 核心抉择 |
| 9 | 550 | 3.5 | 157 | ~190 | 双选 + 陷阱 | 综合 |
| 10 · BOSS | 750 | 3.3 | 227 | ~260 | 烂蒜魔王(多阶段) | 章节 BOSS |

- **难度曲线**：`H` 随关卡上升、`k` 缓慢下降（`k`: 5.0 → 长期 2.0）；每关校验 **最优路线兵力 > N_min 一定裕度、次优路线 < N_min**（制造"要吃对门才过"）。
- 前 3 关不放陷阱门、`N_min` 远低于可得兵力，保证新手必过。
- 关卡耗尽后进入**无尽模式**：程序化生成，`H` 持续升、`k` 逼近下限。

## 6. 胜负判定与星级
- **胜**：击穿所有敌方波次含 BOSS，`N > 0` 抵达终点。
- **负**：任一波 `N < N_min`，兵力归 0。
- **星级**：按**离本关最优解有多近**评定，`ratio = N_peak / targetN`（`N_peak` 为本局峰值兵力，`targetN` 见 §5.1 最优路线兵力）：`ratio ≥ 0.95` → 3★，`≥ 0.75` → 2★，通关即 1★，未通关 0★（阈值可配）。
- 不用旧式 `rem = N_end / N_peak`：`k` 随关卡下降令 `N_min` 吃掉高关绝大部分兵力（关 10 最优解 `rem` 仅 0.13），越会玩星越低，反馈方向是反的。

## 7. 复活机制
- FAIL 后可看**激励视频复活**：在**倒下位置**恢复 `N_revive = max(N_min_of_current_wave, N_peak × 0.5)` 兵力续冲（保证复活后有机会过当前波）。
- 每局复活上限 1–2 次（可配）；超过只能重开。

## 8. 手感参数清单（可热更调参表）
| 参数 | 建议初值 | 说明 |
|---|---|---|
| forwardSpeed | 9 u/s | 世界前进速度 |
| dragSensitivity | 1.0 | 屏幕 X→世界 X 映射系数 |
| followSmooth | 0.15 | 大军跟手平滑（lerp） |
| formationRadiusK | 0.6 | 阵型半径系数（R=k·√N） |
| N_render | 130 | 同屏渲染上限 |
| gateTriggerZ | 门 posZ | 触发判定线 |
| smashShake | 幅 8px / 0.2s | 撞击震屏 |
| bossHitStop | 60 ms | BOSS 受击顿帧 |
| T_min / T_max | 0.6 / 2.5 s | 对撞演出时长夹取 |

## 9. 数据驱动约定
- 门、关卡、敌人、数值曲线（`k(level)`、`H(level)`、升级成本表）全部走 **JSON / 表格配置**，运行时加载；支持远程配置热更（微信小游戏可远程拉取配置）。
- 代码只实现规则引擎（§3.2 / §4.3），数值不写死。

## 10. 玩法相关埋点事件
| 事件 | 关键字段 |
|---|---|
| level_start | level |
| gate_pick | level, gateId, choice（选了哪侧/类型） |
| wave_smash | level, waveIndex, N_before, N_after |
| level_win / level_fail | level, N_peak, N_end, star, failWave |
| revive_show / revive_done | level, reviveCount |
- 门抉择 (`gate_pick`) 与失败波次 (`level_fail.failWave`) 是关卡数值调优的核心数据。
