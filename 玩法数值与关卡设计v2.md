# 《蒜鸟冲冲冲》玩法数值与关卡设计 **v2 · 射击转向**

> **v2 取代 v1**（`玩法数值与关卡设计.md`，对撞模型）。核心循环从「大军对撞消耗血条」改为**「海量小怪迎面冲来 + 自动射击 + 多维倍增门」**。
> v1 的对撞模型（`resolveSmash`/`N_min = H/k`）与 40 关配置**作废**；引擎无关分层、星级框架、养成框架、工程骨架**全部保留**。

## 0. 调研结论 → 硬性设计约束

这些是行业验证过的铁律，设计不得违反：

| 约束 | 依据 | 落到本作 |
|---|---|---|
| **单局 30 秒** | PocketGamer："classic 30 second per level" | `duration ≈ 30s`，赛道长度按前进速度反推 |
| **变化夸张到一眼可见** | "Make the changes hard to miss" | 弹道 1→3 必须满屏弹幕；兵力 ×2 阵型肉眼翻倍 |
| **5 秒内有视觉反馈** | 5 秒是可买量素材的窗口 | **开局 3 秒内首波怪必须出现并被打倒** |
| **难度宽容** | Supersonic："harder to fail and more forgiving" | 漏怪只掉兵不立死；护盾/复活兜底；失败率目标 < 25% |
| **会话 2–3 分钟** | 平均会话时长 | 一次连玩 4–6 关，关间无阻断 |
| **明亮大胆配色** | 降低 CPI | 沿用蒜白 `#F5F0E6` vs 霉绿 `#4A5D2B` 强对比 |

> **素材导向原则**：任何新机制先问「它能不能在 5 秒素材里被看懂」。看不懂的不做。

## 1. 核心循环

```
出兵 N0 → 自动前进 + 自动射击
  → 海量小怪迎面冲来（成群、密集、有压迫感）
  → 蒜瓣子弹成片打倒（击杀飘字 + 尸体堆积）
  → 漏网的怪撞进大军 → 掉兵
  → 穿过倍增门 → 火力/兵力跃升（视觉肉眼可见）
  → 关底 BOSS（高血量单体）
  → 终点倍增区（子弹打倍增板，收尾爽点）→ 结算
```

**主题**：蒜鸟**吐蒜瓣**当子弹打霉菌怪，不另设计武器形象，与「新鲜 vs 霉烂」世界观自洽。

## 2. 火力模型（核心）

### 2.1 四个维度
| 符号 | 含义 | 视觉表现 |
|---|---|---|
| `N` | 兵力：火力基数，同时是被撞掉的「血」 | 阵型大小 |
| `L` | 弹道数：同时能打几个目标（横向覆盖） | 弹幕条数 |
| `R` | 射速：每秒发射次数 | 弹幕密度 |
| `D` | 单发伤害 | 命中特效大小 + 飘字 |

```
单目标 DPS = N × D × R        // BOSS 战只看这个,L 不起作用
总火力    F = L × N × D × R    // 清怪潮看这个
```

### 2.2 为什么这样拆（策略深度的来源）
| 场景 | 该堆什么 | 原因 |
|---|---|---|
| 海量弱怪 | **L** | 打不过来会横向漏网，光堆伤害无用 |
| BOSS 单体 | **D / R** | 只有一个目标，L 完全浪费 |
| 高压怪潮 | **N** | 既是火力基数又是抗撞的血 |

→ 双选门 `×2弹道` vs `×2伤害` 成为**真抉择**：得预判下一段是怪潮还是 BOSS。这是 v1「选数字大的」所没有的。

### 2.3 实现约束（性能）
- 子弹**不逐只生成**：每条弹道是一条「弹流」，用固定节奏的子弹节点 + 对象池；同屏子弹上限 `maxBullets`（建议 120）。
- 怪同屏上限 `maxEnemies`（建议 150），超出用「后排压缩」表现更多。
- 逻辑击杀与视觉击杀解耦：逻辑按 `F` 结算击杀数，视觉挑最近的 N 只播死亡帧（同 v1 池化思路）。

## 3. 敌人与怪流模型

### 3.1 怪流段（关卡的基本单位）
一段怪流由三个参数描述：

| 参数 | 含义 |
|---|---|
| `λ` | 到达速率（只/秒） |
| `h` | 单怪血量 |
| `t` | 持续时间（秒） |

```
该段所需火力   F_min = λ × h
漏网速率       leak  = max(0, λ − F/h)   (只/秒)
本段损兵       loss  = leak × t × c      (c = 每只怪撞掉的兵数)
存活条件       Σloss < N
```

> `F_min = λ × h` 与 v1 的 `N_min = H/k` 同样可推导，关卡照样能设计成「走错门就崩」。

### 3.2 怪的种类（视觉与功能分化）
| 类型 | 特征 | 设计作用 |
|---|---|---|
| 霉菌小怪 | 血低、量大、速度中 | 主体怪潮，考验 `L` |
| 快速孢子 | 血极低、速度快 | 逼玩家提前把火力拉起来 |
| 厚皮烂蒜 | 血高、速度慢 | 考验 `D`，L 高但 D 低会卡住 |
| BOSS 烂蒜魔王 | 单体、超高血、分阶段 | 纯单目标 DPS 检验 |

## 4. 门体系（多种类倍增门）

| 门 | 效果 | 配色 | 说明 |
|---|---|---|---|
| 兵力门 | `N +k` / `N ×2` | 蓝 / 金 | 连排小额 + 稀有大额 |
| 弹道门 | `L +1` / `L ×2` | 紫 | **视觉冲击最强**，素材主力 |
| 射速门 | `R +0.5` / `R ×1.5` | 橙 | 弹幕变密 |
| 伤害门 | `D +1` / `D ×2` | 红 | 飘字变大 |
| 陷阱门 | 任一维 `÷2` 或 `−k` | 黑红 | 制造走位风险 |
| 特殊门（限时 8s） | 穿透弹 / 爆炸弹 / 双倍暴击 | 彩虹 | 短时爽感峰值 |

**排布纪律**
- 每 **8–10 秒**一个门组（单局 3–4 个决策点）。
- **正负同屏**（抄 Gun Rush）：好门旁边一定有坏门，走位才有意义。
- **双选门**两侧都是增益但维度不同（`×2 L` vs `×2 D`），逼玩家判断。
- 门的效果**必须在穿过瞬间有夸张反馈**：弹道数增加时弹幕条数肉眼翻倍 + 音效 + 短震屏。

## 4.5 闸门（射击门）—— 品类标配，参考 Last Z 与「口袋奇兵/三国小小兵」素材

素材里横在路上写着 `621` 的那道门就是它。**不是穿过即生效，而是必须用火力打空数值才能通过。**

| 规则 | 说明 |
|---|---|
| 吃**总火力 F** | 闸门是横跨赛道的单一大目标，L 条弹道全打得上 |
| 打门期间**大军被钉在原地** | 前进停止，但怪流照常涌来 |
| 打门期间**不清怪** | 怪被拦在门后**堆积**，打穿瞬间一起涌出 |
| 血量设计 | `hp = 到达时的 F × 2.5 秒` —— 火力达标者 2.5 秒撕开 |

**为什么它是标配**：这是全局唯一的**纯火力检验点**。够就撕开继续冲，不够就卡在门前被堆积的怪淹掉——惩罚即时可见，比"漏怪掉兵"直观得多。同时它让"火力"这个抽象数值有了具体的对抗对象，也给了 5 秒素材一个天然的高潮镜头。

**实测**：加入闸门后，「一路不动」从能混过 3 关变成**6 关全败**——它把"必须堆火力"这件事变成了硬约束。

## 4.6 敌军密度：按「排」生成，不是零散个体

素材里敌军是**铺满赛道的红色海洋**。故怪流的 `λ` 语义为**排/秒**，每排 `rowSize` 只（7–12）横向铺开 + 轻微抖动。

```
F_min = λ × rowSize × h     // 校验量相应改写
```

**实测**：单局击杀从 296 涨到 700–1100，屏幕上同时存在 120+ 只怪，压迫感与 v1 完全不是一个量级。

## 4.7 门墙：两侧长龙

素材里是**左边一整排 `+1`、右边一整排 `+99`** 排成长龙，而不是零散几个门。抉择在视觉上一眼可见（"贴左吃一排小的，还是绕右吃大的"），也是素材里最抓眼的元素。配置上用 `repeat`（`count` 14–18、`stepZ` 3.2–4）实现。

## 5. 关卡结构与节奏（30 秒切片）

```
0–3s    出兵 + 首波稀疏怪（立刻有击杀反馈，素材窗口）
3–10s   门组 1 → 怪潮 A（考验 L）
10–18s  门组 2（含陷阱）→ 怪潮 B（混入厚皮，考验 D）
18–25s  门组 3（双选）→ 怪潮 C（高压）
25–30s  BOSS 或 终点倍增区 → 结算
```

- **终点倍增区**（抄 Master Gun）：一排 `×2/×3/×5` 倍增板，用子弹打碎它拿金币倍数，爽感集中收尾。
- 每 5 关一个 BOSS 关（结构同 v1）。

| 5 | BOSS 血 1200 | 单目标 DPS 需 ≥180 | 单目标 220 | 小 BOSS | 第一次 L 无用，逼堆 D/R |

## 7. 失败与宽容度（调研要求「更宽容」）

- 漏怪只**掉兵**，不立即失败；`N ≤ 0` 才 FAIL。
- **护盾**：`N` 低于峰值 20% 时给 2 秒无敌闪烁（避免瞬间团灭的挫败）。
- 复活：看激励视频原地续命，恢复 `N = max(当前段 F_min 对应兵力, N_peak × 0.5)`（沿用 v1 §7 思路）。
- 目标失败率 **< 25%**，前 3 关必过。

## 8. 星级（沿用 v1 已验收的公式，换分母）

```
ratio = F_peak / level.targetF     // 峰值总火力 / 该关设计的最优火力
ratio ≥ 0.95 → 3★;  ≥ 0.75 → 2★;  通关 1★;  未通关 0★
```
> 与 v1 同构（原为 `N_peak / targetN`），`starRating` 函数签名不变，只换传入的量。T2 的成果直接复用。

### 8.1 `targetF` 口径（V7 定死，改动需同步 `tools/route.mjs`）

**`targetF` ≡ 「参考最优路线」在完整关卡（含怪流 / 闸门 / 障碍 / 油桶）上实跑出来的 `fPeak`。**

参考最优路线 = **躲开所有障碍 + 打掉所有打得完的桶 + 每道门取火力最大的分支**，
其权威定义是代码而不是文字：`tools/route.mjs` 的 `bestRoute()`，取策略族里**打得过且 fPeak 最高**的那条。

三条推论，配关卡时按这个用：

1. **分母是实跑测出来的，不是公式估的。** 所以 `targetF` 天然可达，不会出现「前段够不着」；
   `ratio` 也不该超过 1.00 —— 超了就说明存在一条比参考路线更强的走法，该把它补进策略族，而不是调分母。
2. **它是实战最优，不是纯贪心。** 拿「不躲障碍、不打桶」当分母的话：躲障碍的人火力反而更高、
   桶奖励走 `applyGate` 又会进 `fPeak`，`ratio` 必然破线（V6 实测关 9 = 107%、关 12 = 137%，3★ 变白送）。
3. **`targetF` 只由 `tools/rebalance.mjs` 回填，不手填。** 手填必然与配置漂移 ——
   V6 的 `targetF` 是拿「摘掉怪流闸门、却留着障碍和桶」的脏空跑算的：障碍掉兵压低了分母、
   桶奖励抬高了实跑分子，两头一夹就是那个 137%。

配套护栏（`prototype-v2/src/core/rules.js`，与手感旋钮分开放，属配置纪律）：

| 常量 | 值 | 管什么 |
|---|---|---|
| `OBSTACLE_MIN_CLEARANCE` | 0.5 | 任一相位都得有一条 lane 留得出余量，否则是「必中」配置 |
| `BARREL_REWARD_CAP` | 0.15 | 单桶收益上限，量的是**拾取那一刻**的火力占比 |
| `BARREL_REWARD_FLOOR` | 0.08 | 单桶收益下限，低于它玩家学到的是「别理桶」 |

桶的收益率按 `barrel.pickStats`（参考最优路线实跑到该桶时的属性快照，由 `rebalance.mjs` 写回配置）计算。
**不能拿起手属性估**：`add` 类奖励的收益率是 `value/dim`，与拾取时该维多大成反比 ——
同一个 `N+2`，起手 N=15 是 13%，实跑拾到时 N=70 只有 2.9%，差 5 倍。

## 12. 与 v1 的差异汇总（给执行 session）

**作废**：`resolveSmash`/`smashDuration`/`buildWaves` 对撞模型、`levels.json` 全部 40 关、HUD 的敌方血条语义。
**保留**：引擎无关分层、`ObjectPool`、`unitFormationSlot`、`starRating`（换分母）、`Progress` 全套、Cocos 工程与构建管线、UI 框架、美术主体。
**新增**：射击/弹道/怪流模型、门的四维体系、终点倍增区、护盾宽容机制。
**待补美术**：蒜瓣子弹、枪口/吐弹特效、小怪的小体型海量变体、弹道/射速/伤害三种新门框配色。
