# 《蒜鸟冲冲冲》每日签到 / 任务 / 分享裂变系统设计（P1）

> 留存 + 传播系统，纯 IAA 阶段即可做（不涉及内购）。数据本地存 + 微信云备份，全部配置表驱动，复用 `AdManager`（广告spec）、`Analytics`、`Storage`、`EventBus`（工程骨架）。

## 1. 每日签到
### 玩法
- 7 天循环签到，奖励逐日递增，第 7 天大奖（皮肤碎片 / 免费复活券）。
- **断签**：默认重置到第 1 天；提供**补签**——看激励视频补回昨日（呼应变现）。
### 数据
```ts
SignData { lastSignDate: string; streak: number; claimedToday: boolean }
```
- 判定：对比 `lastSignDate` 与今日日期；跨日重置 `claimedToday`，断签重置 `streak`。
### 配置
```json
signRewards: [
  {"day":1,"type":"coin","v":100}, {"day":2,"type":"coin","v":150},
  {"day":7,"type":"skin_frag","v":10}
]
```
### 流程
领取 → 发奖（`Storage` 加金币/碎片）→ 埋点 `sign_claim` → 更新 `SignData`。

## 2. 每日任务
### 玩法
- 每日 3–5 个任务，各自给金币/碎片；**全部完成给额外宝箱**。
- 任务示例：通关 3 关、单局兵力达 500、吃门 50 次、看 1 次广告、分享 1 次。
### 进度驱动（关键：挂玩法事件）
- 通过 `EventBus` 订阅玩法埋点事件更新进度，无需侵入玩法代码：

| 任务 | 监听事件（玩法文档 §10 / 广告spec §6） |
|---|---|
| 通关 N 关 | `level_win` |
| 单局兵力达标 | `level_win.N_peak` |
| 吃门 N 次 | `gate_pick` |
| 看广告 | `ad_reward` |
| 分享 | `share_click` |

### 数据
```ts
DailyTask { id: string; target: number; progress: number; claimed: boolean }
TaskData  { date: string; tasks: DailyTask[]; allClaimed: boolean }
```
- 刷新：每日 0 点或当日首次登录重置，从任务池抽 N 个（可固定核心 + 随机）。
### 配置
```json
dailyTaskPool: [
  {"id":"clear3","desc":"通关3关","ev":"level_win","target":3,"reward":{"coin":200}},
  {"id":"troop500","desc":"单局兵力达500","ev":"level_win","field":"N_peak","target":500,"reward":{"coin":150}},
  {"id":"ad1","desc":"看1次广告","ev":"ad_reward","target":1,"reward":{"skin_frag":2}}
]
```

## 3. 分享裂变
### 3.1 分享场景
| 场景 | 触发点 | query |
|---|---|---|
| 晒战绩 | 通关结算 | `act=result&lv&score` |
| 挑战好友 | 通关 / 榜单 | `act=challenge&lv&score&from`（见排行榜spec §4.2） |
| 求复活 | FAIL 页 | `act=revive_help&lv` |
| 每日分享任务 | 任务 / 主界面 | `act=daily` |

### 3.2 分享奖励与微信的坑（重要）
- **微信已取消分享成功回调**：`wx.shareAppMessage` **无法可靠判断是否分享成功 / 被点击**。
- 因此奖励只能设计成 **"点击分享即发"** 或 **"每日首次分享发"**，接受一定薅羊毛，用**每日限次**防刷。
- **深度裂变**（好友助力复活 / 助力开箱，需知道"谁帮了我"）**需服务器，或用分享 query + 被邀者托管数据间接实现** → **P1 先做轻量分享奖励，助力类后置**。
### 3.3 数据
```ts
ShareData { date: string; sharedCountToday: number; dailyRewardClaimed: boolean }
```

## 4. 系统联动
- **广告**：签到补签、任务"看广告"、求复活 → 复用 `AdManager`（广告spec）。
- **埋点**：`sign_claim`、`task_progress`、`task_claim`、`share_click(scene)`。
- **存档**：`SignData` / `TaskData` / `ShareData` 进 `Storage`（本地 + 云备份）。
- **UI**：主界面签到入口（红点）、任务面板（进度条/领取/全清宝箱）、各场景分享按钮。

## 5. 防刷与边界
- 签到 / 任务 / 分享奖励**均每日限次**。
- **改设备时间刷签到**：P1 用本地日期，接受该风险；关键奖励（高价值）后续加**服务器校时**校验（后置增量）。
- 分享薅羊毛：靠每日限次 + 奖励价值控制，纯 IAA 阶段可接受。

## 6. 任务拆解（P1）
| 编号 | 任务 | 依赖 | 验收 | 人天* |
|---|---|---|---|---|
| S01 | 签到数据 + 配置 + 领取逻辑 | Storage | 7 天签到 / 断签 / 补签正确 | 1 |
| S02 | 签到 UI（红点 / 领取动画） | S01 | 完整签到流程 | 1 |
| S03 | 每日任务：进度事件驱动引擎 | EventBus, Analytics | 进度随玩法事件更新、跨日刷新 | 1.5 |
| S04 | 任务 UI + 领取 + 全清宝箱 | S03 | 面板与领取正确 | 1 |
| S05 | 分享封装（scene + query + 奖励 + 限次） | AdManager | 各 scene 分享并发奖 | 1 |
| S06 | 分享 UI 入口 + 每日分享任务串联 | S05, S03 | 分享得奖、与任务联动 | 0.5 |
| S07 | 埋点接入 | 全部 | 事件齐 | 0.5 |

\* 单人粗估，约 **6.5 人天**。
