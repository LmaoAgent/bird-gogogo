# 《蒜鸟冲冲冲》美术需求清单

> 本清单是**美术资源规格说明（spec）**，字段结构化，既可交付真人美术，也可直接喂给 AI 生成工具（codex）按条目产出资源。描述「要什么、什么规格、什么优先级」，不锁定产出手段。

## 0. 技术基准与全局规范

| 项 | 规范 |
|---|---|
| 平台 | 微信小游戏，竖屏，引擎 Cocos Creator |
| 设计分辨率 | 1080×1920（竖屏基准），适配安全区 |
| 资源形态【建议】 | **2D 精灵为主**；角色用骨骼动画（Spine / DragonBones），场景/门用图片，特效用粒子或帧序列。轻量 3D 为备选路线（备选时角色出低模+贴图） |
| 格式 | PNG（带透明通道 RGBA）；打包 SpriteAtlas 图集（.plist/.json）；@2x 出图 |
| 包体约束 | 首包 ≤ 4MB，非首发资源（皮肤、后续关卡场景）走分包 / 远程加载 |
| 命名规范 | `模块_对象_状态_序号`，全小写下划线，如 `hero_garlicbird_run_01.png`、`enemy_moldking_roar.json` |
| 风格基调 | 卡通魔性、高饱和、粗描边、大对比、小屏可辨识；集群识别优先 |
| 主色对立 | **新鲜蒜白 `#F5F0E6` / 蒜紫皮 `#C9B8D6`** vs **霉烂墨绿 `#4A5D2B` / 腐黑 `#2B2420`**；点缀橙黄嘴 `#F2A83B`、蒜芽绿 `#8FBF4D` |

> 若用 codex / AI 程序化生成：角色可先出**静态多状态帧或 SVG 占位**，骨骼动画后补；每个条目已给出用途与规格，可按 P0 优先级逐条生成。

## 1. 主角：蒜鸟大军（P0 · 最高优先）

### 1.1 基础形象设定（base 蒜鸟）
- **身体**：白胖独头蒜造型，纵向蒜瓣分割纹路，圆润饱满。
- **头部**：短尖鸟嘴（橙黄），圆眼（眼神是灵魂，带"算了/摆烂"的呆萌无语感），头顶可有一小撮蒜芽。
- **四肢**：小短腿（跑动）、小翅膀（摆动）。
- **比例**：Q 版 2–2.5 头身，远看也要能认出是"蒜+鸟"。
- **交付**：正/侧/背三视图 + 表情差分（呆萌 / 惊慌 / 得意 / 无语 至少 4 种）。
- **⚠️ 版权要求（形象来源＝AI 生成 + 原创化）**：必须产出**原创**蒜鸟形象——自定特征比例、配色、表情规范，**不得与特定 meme 作者的作品高度相似**；保留生成参数与迭代过程记录，以备权属证明。

### 1.2 动画状态（骨骼动画）
| 状态 | 说明 | 循环 | 优先级 |
|---|---|---|---|
| run | 跑步（核心） | ✔ | P0 |
| cheer | 欢呼（通关） | ✔ | P0 |
| panic / die | 惊慌被消灭（FAIL / 减兵） | ✖ | P0 |
| boost | 加速冲刺（吃门 / 复活） | ✔ | P1 |
| idle | 待机（主界面） | ✔ | P1 |

### 1.3 集群渲染要求
- 大军是**几十到上百只同屏**，需低开销：**同一套骨骼动画实例化**，通过 `tint` 微调色相、`scale` 微调大小、`phase` 错开动画相位制造丰富度，**不逐只单独绘制**。
- 提供 1 套 base 即可满足集群，差异靠程序参数扰动。

### 1.4 皮肤变体（P1 · 内购 / 解锁供给，复用 base 骨骼仅换贴图）
| 皮肤 | 视觉 | 稀有度 | 优先级 |
|---|---|---|---|
| 独头蒜（金） | 金色发光蒜身 | 稀有 | P1 |
| 黑蒜 | 暗黑油亮 | 稀有 | P1 |
| 糖蒜 | 半透明 Q 萌琥珀色 | 普通 | P1 |
| 发芽蒜 | 头顶长蒜苗 | 普通 | P1 |
| 腊八蒜 | 青绿·节日 | 节日 | P2 |

## 2. 敌人：霉烂军团（P0）

### 2.1 杂兵（集群/洪流，与蒜白对立的墨绿霉烂质感）
| 名称 | 说明 | 优先级 |
|---|---|---|
| 霉菌怪 | 绿毛孢子小怪，构成敌方洪流主体 | P0 |
| 蒜蛆虫 | 蠕动虫形杂兵 | P1 |
| 洋葱刺客 | 催泪 debuff（可选机制） | P2 |
- 动画：`flow`（洪流蠕动，循环）、`die`（被撞消灭，喷霉斑/蒜末）。

### 2.2 BOSS：烂蒜魔王（P0 · 首发 1 个）
- **造型**：巨型发黑腐烂蒜，流脓、霉斑、獠牙，体量有压迫感（对应截图红巨人位）。
- **多阶段**：血条分段，phase2 狂暴变色。
- 动画：`idle`、`roar`（登场）、`hit`（受击顿帧）、`phase2`（狂暴）、`die`。
- 与顶部**血条 UI** 联动（血条见 §6）。
- 后续版本追加精英怪 / 其它 BOSS。

## 3. 场景（P0 · 1 套；P1 · 追加 2–3 套换肤）
- **结构**：桥 / 赛道，三段式布局——左加成区 / 中赛道 / 右加成区（延续截图），**可无限循环平铺拼接**（跑酷需要）。
- **首发主题（已定）：菜市场大桥**（蒜味最足、生活化，买量素材辨识度高）；腌菜坛通道、冷链冰库作为后续换肤。
- 资源拆分：赛道地面（tile 循环）、两侧护栏、背景远景（天空 / 桥塔 / 水面）、氛围道具（挂蒜串、菜筐等）。
- P1 追加：腌菜坛通道、冷链冰库等换肤主题。

## 4. 加成门 / 陷阱门（P0 · 玩法核心视觉）
| 门类型 | 视觉 | 备注 |
|---|---|---|
| 蓝色 `+1` 连排门 | 蓝色半透明门框 | 一长串累加 |
| 黄色 `+99` 大门 | 金黄发光门框 | 高回报 |
| 乘法门 `×2/×3` | 紫色 / 彩虹发光 | 强调"翻倍" |
| 陷阱门 `-50 / ÷2` | 红黑危险配色 | 惩罚 |
| 双选门 | 左右并排、色彩强对比 | 抉择点 |

- **门框做图（4 种配色 base + 可拉伸），门上数字与符号用程序渲染**——需一套**醒目位图数字字体**：`0–9`、`+ - × ÷`（P0，两套配色：加成绿/白、陷阱红）。
